# スケジュール月表示の改善

## 実装内容

### 1. 「他N件」クリック時の詳細表示機能

月表示の「個人」ビューで、複数のタスクがある日に「他N件」と表示される部分をクリックすると、その日のタスク一覧がモーダルで開くようになりました。

#### 変更点
- 詳細モーダルの表示条件を `calendarViewMode === 'all'` から削除
- 個人モード（`individual`）でも詳細モーダルが表示されるように変更
- Googleマップのような直感的なUIを実現

#### 動作
- 「他N件」ボタンをクリック → その日のスケジュール一覧がモーダルで表示
- モーダル内でユーザーごとにグループ化されたスケジュールを確認可能
- 各スケジュールをクリックすると詳細編集モーダルが開く

### 2. 日をまたぐタスクの表示最適化

日をまたぐタスクがない日は、上に詰めて表示されるようになりました。

#### 変更点
- 各日ごとに、その日に実際に表示される複数日バーの数を計算する関数 `getLanesForDay()` を追加
- 週全体で一律の高さを確保するのではなく、各日に必要な高さだけスペースを確保
- 日をまたぐタスクがない日は、複数日バーのスペースが0になり、単日スケジュールが上に詰まって表示される

#### 実装詳細
```typescript
// 各日ごとに必要なレーン数を計算
const getLanesForDay = (dayCol: number): number => {
  let maxLane = 0;
  lanes.forEach((lane, laneIndex) => {
    const hasBarOnDay = lane.some((bar) => bar.startCol <= dayCol && bar.endCol >= dayCol);
    if (hasBarOnDay) maxLane = Math.max(maxLane, laneIndex + 1);
  });
  return maxLane;
};

// この日に表示される複数日バーの数を計算
const lanesForThisDay = getLanesForDay(dayIndex);
const multiDayBarHeight = lanesForThisDay * (BAR_HEIGHT + BAR_GAP);
```

#### 効果
- 日をまたぐタスクがある週でも、該当しない日は空白が発生しない
- カレンダーの表示がより効率的で見やすくなる
- スペースの無駄が削減される

## 変更ファイル

- `frontend/src/pages/Schedule.tsx`
  - 詳細モーダルの表示条件を変更
  - `getLanesForDay()` 関数を追加

## テスト項目

1. 月表示の個人モードで「他N件」をクリックして詳細モーダルが開くことを確認
2. 詳細モーダル内でスケジュールをクリックして編集モーダルが開くことを確認
3. 日をまたぐタスクがある週で、該当しない日が上に詰まって表示されることを確認
4. 日をまたぐタスクがない週で、すべての日が上に詰まって表示されることを確認
5. 複数の日をまたぐタスクが重なる場合でも、正しくレーン配置されることを確認
