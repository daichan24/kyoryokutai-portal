# 週境界計算の仕様

## 概要

SNS投稿管理システムでは、**月曜日 9:00 JST** を週の開始時刻として使用します。

## 週の定義

```
週の開始: 月曜日 09:00 JST (= 月曜日 00:00 UTC)
週の終了: 翌月曜日 09:00 JST (= 翌月曜日 00:00 UTC)
```

### 例

| 日時（JST） | 所属する週 |
|------------|-----------|
| 月曜 08:59 | 前の週 |
| 月曜 09:00 | 今の週（開始） |
| 日曜 23:59 | 今の週 |
| 翌月曜 09:00 | 次の週 |

## 週キー形式

週キーは `YYYY-WNN` 形式で表現されます。

```
2024-W01  → 2024年第1週
2024-W03  → 2024年第3週
2024-W53  → 2024年第53週（2024年は53週ある）
```

**注意**: 週キーの年は週の開始日（月曜日）の年を使用します。
例えば、2025年1月1日（水曜日）は2024年12月30日（月曜日）の週に属するため、週キーは `2024-W53` になります。

## タイムゾーン変換

サーバーはUTCで動作しますが、週境界はJST（UTC+9）で計算します。

```
JST 09:00 = UTC 00:00
```

日本は夏時間（DST）がないため、常に UTC+9 で固定です。

### YYYY-MM-DD 形式の日付入力

日付のみ（時刻なし）の入力は、**JST 正午（12:00）** として解釈されます。

```
入力: "2024-01-15"
解釈: 2024-01-15 12:00 JST = 2024-01-15 03:00 UTC
```

これにより、月曜日の日付を入力した場合でも、9:00 JST の週境界を正しく判定できます。

## 実装

週境界計算は `backend/src/utils/weekBoundary.ts` に実装されています。

```typescript
import { getWeekBoundaryForDate, getCurrentWeekBoundary } from './utils/weekBoundary';

// 特定の日時の週境界を取得
const { weekStart, weekEnd, weekKey } = getWeekBoundaryForDate(new Date('2024-01-15T03:00:00Z'));
// weekStart: 2024-01-15T00:00:00.000Z (月曜 09:00 JST)
// weekEnd:   2024-01-22T00:00:00.000Z (翌月曜 09:00 JST)
// weekKey:   "2024-W03"

// 現在の週境界を取得
const { weekStart, weekEnd, weekKey } = getCurrentWeekBoundary();
```

## よくある質問

**Q: 月曜日の朝8時に投稿した場合、どの週に属しますか？**

A: 前の週に属します。週の開始は月曜日 9:00 JST なので、8:59 JST までは前の週です。

**Q: 週キーの週番号はISO 8601と同じですか？**

A: いいえ。このシステムの週番号は月曜日 9:00 JST 起算の独自計算です。ISO 8601（木曜日基準）とは異なる場合があります。

**Q: 年をまたぐ週はどう扱われますか？**

A: 週の開始日（月曜日）の年が週キーの年になります。例えば、2024年12月30日（月曜）〜2025年1月5日（日曜）の週は `2024-W53` です。
